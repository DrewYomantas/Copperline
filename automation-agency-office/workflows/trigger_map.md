# Trigger Map

| Trigger Event | Event Source | Triggered Agent(s) | Expected Action | Escalation Path |
|---|---|---|---|---|
| new email reply | Inbox / email connector | `pipeline_manager_agent`, `outreach_draft_agent` | Update lead state, draft next response option, log follow-up timer. | If unanswered beyond SLA, escalate to `revenue_operations_supervisor_agent`, then founder for repeated misses. |
| new social DM | Social platform inbox | `social_engagement_agent`, `pipeline_manager_agent` | Draft compliant reply, classify intent, update lead/pipeline if sales-related. | If no response within SLA, escalate to `revenue_operations_supervisor_agent`; founder if policy-sensitive. |
| new social comment | Social platform comments stream | `social_engagement_agent`, `content_creation_agent` | Draft comment response and capture recurring FAQ/topic for content backlog. | Escalate sensitive or reputational issues to founder immediately. |
| new lead added to `prospects.csv` | Prospect intake workflow / file update | `business_research_agent`, `outreach_draft_agent`, `pipeline_manager_agent` | Research account, queue personalized outreach draft, initialize pipeline tracking record. | If ICP mismatch or data quality issues recur, escalate to founder via `revenue_operations_supervisor_agent`. |
| lead moved to `proposal_needed` state | Pipeline state change in `leads_pipeline.csv` | `proposal_generator_agent`, `pipeline_manager_agent` | Draft one-page proposal using approved template and in-range pricing; track due date. | Any pricing exception or scope outside v1 -> founder approval required immediately. |
| new signed project added to `active_projects.md` | Project onboarding update | `solution_builder_agent`, `deployment_agent`, `qa_validation_agent` | Create implementation plan, stage deployment tasks, prepare QA checklist/gates. | Access blockers, scope conflicts, or timeline risk escalate to founder same day. |
| automation failure detected | Monitoring logs / runtime alerts | `automation_health_monitor_agent`, `maintenance_response_agent`, `qa_validation_agent` | Open incident, classify severity, assign remediation tasks, validate fix after changes. | Critical client impact -> immediate founder escalation; track in incident and health logs. |
| unresolved incident exceeding SLA | `incident_log.md` + SLA checker | `maintenance_response_agent`, `revenue_operations_supervisor_agent`, `chief_of_staff_agent` | Re-prioritize incident, update owners/ETA, force escalation review. | If still unresolved after next SLA window, escalate to founder and mark in weekly brief. |
| stale pipeline stage | `leads_pipeline.csv` inactivity threshold | `pipeline_manager_agent`, `revenue_operations_supervisor_agent` | Trigger follow-up task, update stage notes, flag at-risk opportunities. | Repeated stale stages escalate to founder with process-correction plan. |
| stale project stage | `project_status_log.md` inactivity threshold | `revenue_operations_supervisor_agent`, `solution_builder_agent`, `deployment_agent` | Identify blocker, assign owner/action, update project timeline notes. | Missed delivery risk -> same-day founder escalation. |
| missing weekly report | Weekly check for `weekly_summary.md` and `ceo_brief.md` | `chief_of_staff_agent`, `knowledge_capture_agent` | Generate missing report artifacts and log reason for delay. | If still missing by next business day, escalate to founder. |
| office inactivity threshold breached | Office telemetry / no meaningful updates in logs | `revenue_operations_supervisor_agent`, `chief_of_staff_agent` | Run health sweep, identify dormant areas, create recovery task list. | If inactivity persists >24h after intervention, escalate to founder. |
